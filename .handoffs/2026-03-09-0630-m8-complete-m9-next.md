## Decisions made

- **CST uses Box allocation** (not arena) — simpler, profileable later. Design at `docs/plans/2026-03-09-cst-design.md`.
- **Zero-length spans = virtual events** — CstBuilder skips whitespace insertion and offset tracking for virtual tokens (BlockEnd, implicit DocumentStart).
- **Implicit empty scalars skipped in CST** — plain/empty scalars borrow the next token's span, which would cause double source text extraction. Detected via `value.is_empty() && style == Plain`.
- **Out-of-order comments handled by span check** — if `span.start < last_offset`, the comment's source region is already in a whitespace gap. Skip to avoid duplication.
- **12 flow close token cases allowlisted** — `]`/`}` appears twice in CST output for complex flow nesting. Root cause: MappingEnd/SequenceEnd for flow collections have non-zero spans overlapping with whitespace gaps. Fix requires tracking which source bytes are "claimed" by events.
- **Event queue in parser** — VecDeque replaces recursive parse_next calls. Structural events (BlockEntry, KeyIndicator, ValueIndicator) are returned directly; empty scalars queued. This avoids event ordering issues between queued comments and returned events.

### Performance tuning (deferred — needs profiling)

Benchmark snapshot (comparative, 10K records ~1.2MB):
- yamalgam scan: 42ms (3rd — faster than saphyr, slower than rust_yaml/libyaml)
- yamalgam parse: 72ms (slowest — rust_yaml does full parse in 30ms!)
- Scanner hot spots: quoted scalar allocation (`String::new()` + char-by-char push for simple `"record-N"` strings), per-character whitespace skipping
- Parser hot spots: `Box<dyn Iterator>` virtual dispatch for token source, event queue overhead

Recommended optimizations (in priority order):
1. **Zero-copy quoted scalars** (`scanner.rs:1970,2127`) — track `needs_owned`, borrow from input when no escapes/folds. Covers 10K strings in the benchmark.
2. **Bulk whitespace skipping** — `scan_to_next_token` advances character-by-character; memchr/chunk-based skip would help.
3. **Generic parser token source** — replace `Box<dyn Iterator>` with a generic type parameter to eliminate virtual dispatch on the hot path.
4. **Profile first** — `cargo instruments` or `samply` flamegraph before further work.

## Landmines

- **CST flow close duplication** — 12 YAML Test Suite cases produce extra `]`/`}` in CST output. Root cause is flow collection end events (MappingEnd, SequenceEnd) having non-zero spans that overlap with content already captured in whitespace gaps. Allowlisted in `crates/yamalgam-compare/tests/cst_round_trip.rs`. Fix requires a "span claiming" mechanism in the CstBuilder.
- **Parser event ordering** — comments may arrive AFTER MappingEnd/SequenceEnd in the event stream (queued by `collect_comments` during `peek_token`). The CstBuilder handles this by checking `span.start < last_offset` and skipping. But this means some comments end up in whitespace tokens rather than Comment tokens in the CST.
- **Empty scalar span borrowing** — the parser's `emit_empty_scalar()` uses the NEXT token's span (e.g., empty key before `:` gets the Value token's span). The CstBuilder detects this via `value.is_empty() && style == Plain` and skips source text extraction. If the parser changes how it creates empty scalars, the CST builder may break.
- **`cargo-deny` CI failure** — the `cargo-deny` job fails because MSRV 1.94.0 isn't installed on the CI runner. This is a pre-existing issue. PR #81 was admin-merged past it. Other checks (lint, test, msrv) all pass.
